## Overview

Ollama allows you to run large language models locally on your machine. This means:

- **Free**: No API costs
- **Private**: Your data never leaves your computer
- **Fast**: No network latency
- **Offline**: Works without internet connection

Perfect for development, testing, or privacy-sensitive projects.

## Prerequisites

<Steps>
  <Step title="Install Ollama">
    Download and install Ollama from [ollama.ai](https://ollama.ai):
    
    <CodeGroup>
    ```bash macOS
    # Download from ollama.ai or use Homebrew
    brew install ollama
    ```
    
    ```bash Linux
    curl -fsSL https://ollama.ai/install.sh | sh
    ```
    
    ```bash Windows
    # Download installer from ollama.ai
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Start Ollama Server">
    ```bash
    ollama serve
    ```
    
    The server will start on `http://localhost:11434`
  </Step>
  
  <Step title="Pull a Model">
    Download a model (e.g., Llama 3.2):
    
    ```bash
    ollama pull llama3.2
    ```
    
    <Info>
      First-time download may take a few minutes depending on model size.
    </Info>
  </Step>
  
  <Step title="Install ScrapeGraphAI">
    ```bash
    pip install scrapegraphai
    playwright install
    ```
  </Step>
</Steps>

## Basic Configuration

```python
from scrapegraphai.graphs import SmartScraperGraph

graph_config = {
    "llm": {
        "model": "ollama/llama3.2",
        "temperature": 0,
        "model_tokens": 4096,
    },
    "verbose": True,
    "headless": False,
}

smart_scraper_graph = SmartScraperGraph(
    prompt="Find some information about the founders.",
    source="https://scrapegraphai.com/",
    config=graph_config,
)

result = smart_scraper_graph.run()
print(result)
```

<Note>
  This example is from: `examples/smart_scraper_graph/ollama/smart_scraper_ollama.py`
</Note>

## Configuration Options

### Custom Base URL

If Ollama is running on a different host or port:

```python
graph_config = {
    "llm": {
        "model": "ollama/llama3.2",
        "base_url": "http://192.168.1.100:11434",  # Remote Ollama server
        "temperature": 0,
    },
}
```

### JSON Format Mode

Force JSON output (required for some models):

```python
graph_config = {
    "llm": {
        "model": "ollama/mistral",
        "temperature": 0,
        "format": "json",  # Ollama needs format specified
    },
}
```

<Warning>
  Some Ollama models require `"format": "json"` for structured output. If you get parsing errors, add this option.
</Warning>

### Embeddings Configuration

Use local embeddings for better RAG performance:

```python
graph_config = {
    "llm": {
        "model": "ollama/llama3.2",
        "temperature": 0,
    },
    "embeddings": {
        "model": "ollama/nomic-embed-text",
        "temperature": 0,
    },
}
```

```bash
# Pull the embedding model
ollama pull nomic-embed-text
```

## Performance Tips

<AccordionGroup>
  <Accordion title="Use GPU Acceleration">
    Ollama automatically uses GPU if available. Verify with:
    
    ```bash
    ollama ps
    ```
    
    For NVIDIA GPUs, ensure CUDA is installed. For Apple Silicon, Metal is used automatically.
  </Accordion>
  
  <Accordion title="Increase Context Length">
    For long documents, increase `model_tokens`:
    
    ```python
    "llm": {
        "model": "ollama/llama3.2",
        "model_tokens": 128000,  # Maximum context
    }
    ```
  </Accordion>
  
  <Accordion title="Keep Model in Memory">
    Ollama keeps models in memory for 5 minutes by default. Increase this:
    
    ```bash
    # Set to 1 hour
    export OLLAMA_KEEP_ALIVE=1h
    ollama serve
    ```
  </Accordion>
  
  <Accordion title="Use Lighter Models for Simple Tasks">
    For basic scraping, use smaller models:
    
    ```python
    "model": "ollama/llama3.2:1b"  # Fast and efficient
    ```
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Connection Refused">
    **Error**: `Connection refused to http://localhost:11434`
    
    **Solution**: Ensure Ollama is running:
    ```bash
    ollama serve
    ```
  </Accordion>
  
  <Accordion title="Model Not Found">
    **Error**: `model 'llama3.2' not found`
    
    **Solution**: Pull the model first:
    ```bash
    ollama pull llama3.2
    ```
  </Accordion>
  
  <Accordion title="Out of Memory">
    **Error**: System runs out of RAM
    
    **Solution**: Use a smaller model:
    ```python
    "model": "ollama/llama3.2:1b"  # Only 2GB RAM
    ```
  </Accordion>
  
  <Accordion title="JSON Parsing Error">
    **Error**: Failed to parse JSON response
    
    **Solution**: Add `format` parameter:
    ```python
    "llm": {
        "model": "ollama/mistral",
        "format": "json",  # Force JSON output
    }
    ```
  </Accordion>
</AccordionGroup>
